# Daily Journal – 6 July 2026


## Objective 

Conduct research on different approaches for implementing a web crawler capable of extracting data from modern websites, including JavaScript-rendered pages.

## Work Performed

### Research on Web Data Fetching Techniques

The primary focus of today's work was to explore various methods of fetching data from websites for the web crawler implementation.

#### 1. Fetching HTML Using libcurl

I began by researching the use of **libcurl** for downloading webpage content. Through experimentation, I found that libcurl is capable of fetching the HTML source of a webpage efficiently. However, it only retrieves the **static HTML** returned by the server.

This approach works well for traditional websites but is insufficient for modern web applications where most of the content is generated dynamically using JavaScript.

#### 2. Identifying Solutions for Dynamic Websites

After identifying the limitation of libcurl, I researched different techniques to handle JavaScript-rendered websites. Three primary approaches were explored:

* **Direct API Requests**

  Investigated whether the required data could be obtained by directly calling the backend APIs used by the website.
  Studied how browser developer tools can be used to inspect network requests and identify API endpoints.

* **Browser Automation (Puppeteer and Similar Tools)**

   Explored Puppeteer and other browser automation frameworks capable of launching a real browser, executing JavaScript, and returning the fully rendered HTML.
   Studied how browser automation can be integrated with a C++ web crawler to scrape dynamic content.

* **Chromium Embedded Framework**

  Researched Chromium Embedded Framework (CEF) as another solution for rendering JavaScript-heavy websites.
Studied how CEF embeds a full Chromium browser within a C++ application.
Compared its capabilities with Puppeteer and identified that CEF offers tighter integration with native C++ applications, although it introduces greater complexity and resource usage.

## Key Learnings

* **libcurl** is effective for downloading static webpage content but cannot execute JavaScript.
* Modern websites often load their content dynamically, making static HTML insufficient for complete data extraction.
* Browser automation tools provide a practical solution by rendering pages before extracting content.
* In many cases, directly consuming a website's backend APIs can be more efficient than scraping rendered HTML.

## Outcome

By the end of the research, I had a clear understanding of the strengths and limitations of different web data extraction techniques. This helped identify multiple implementation strategies for supporting both static and JavaScript-driven websites in the web crawler project, along with their respective trade-offs.
